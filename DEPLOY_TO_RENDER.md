# 🚀 Deploy HashNHedge Unified Backend to Render

## Overview

This guide will help you deploy the **consolidated HashNHedge backend** to Render.com, replacing your old separate services with one unified API.

---

## ✅ What You're Deploying

### Unified Server (`api/server-unified.js`)
- **API Server** (Community, Vendor, Worker endpoints)
- **Mining Pool** (Worker registration, shares, payments)
- **Network Stats** (Real-time statistics)
- **Database** (PostgreSQL via Prisma)

### Replaces
- ❌ Old `hashnhedge-api` service
- ❌ Old `hashnhedge-pool` service
- ✅ Now: **One unified service**

---

## 📋 Pre-Deployment Checklist

### 1. Database Information
You need your Render PostgreSQL connection string:
```
postgresql://hashnhedge_api_user:PASSWORD@HOST/hashnhedge_api
```

**Find it:**
1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click on your PostgreSQL database: `hashnhedge-db`
3. Copy "Internal Database URL" or "External Database URL"

### 2. Environment Variables Ready
Have these ready:
- ✅ `DATABASE_URL` (from step 1)
- ✅ `OFFICIAL_WALLET_ADDRESS` (your public Solana wallet)
- ⚠️ `ADMIN_API_KEY` (auto-generated by Render)
- ⚠️ `SESSION_SECRET` (auto-generated by Render)
- ⚠️ `JWT_SECRET` (auto-generated by Render)

## 🚀 Deployment Methods

### Method 1: Using Render Dashboard (RECOMMENDED)

#### Step 1: Prepare Repository
```bash
cd /c/Users/gnul/Desktop/hashnhedge-consolidated

# Make sure unified server is in place
ls api/server-unified.js

# Commit changes
git add .
git commit -m "Add unified backend server"
git push origin main
```

#### Step 2: Create New Web Service on Render

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click **"New +" → "Web Service"**
3. Connect your GitHub repository: `knol3j/HNH`
4. Configure:

**Basic Settings:**
```
Name: hashnhedge-unified-api
Region: Oregon (US West)
Branch: main
Root Directory: (leave blank)
Environment: Node
Build Command: npm install && npx prisma generate && npx prisma migrate deploy
Start Command: node api/server-unified.js
Plan: Free
```

#### Step 3: Add Environment Variables

Click **"Environment"** tab and add:

**Required:**
```
NODE_ENV=production
API_PORT=10000
PORT=10000
DATABASE_URL=<paste your Render PostgreSQL URL>
DATABASE_URL_UNPOOLED=<paste your Render PostgreSQL URL>
OFFICIAL_WALLET_ADDRESS=<your public Solana wallet>
```

**Auto-Generate (Click "Generate Value"):**
```
ADMIN_API_KEY
SESSION_SECRET
JWT_SECRET
```

**Optional:**
```
ALLOWED_ORIGINS=https://hashnhedge.com,https://www.hashnhedge.com
POOL_FEE_AI=0.30
POOL_FEE_MINING=0.03
MIN_PAYOUT=0.01
SOLANA_NETWORK=mainnet-beta
```

#### Step 4: Deploy
1. Click **"Create Web Service"**
2. Wait for build (3-5 minutes)
3. Check logs for errors
4. Visit your service URL: `https://hashnhedge-unified-api.onrender.com`

---

## 🐛 Troubleshooting

### Issue: Build Fails
**Check:**
- `package.json` has all dependencies
- `prisma` is in dependencies
- Build command is correct

**Fix:**
```bash
npm install
npm run build
```

### Issue: Database Connection Fails
**Check:**
- `DATABASE_URL` is correct
- Database is running on Render
- IP whitelist allows Render (should be automatic)

**Fix:**
1. Go to Render PostgreSQL dashboard
2. Copy "Internal Database URL"
3. Update `DATABASE_URL` in environment variables

### Issue: 503 Service Unavailable
**Check:**
- Server started successfully (check logs)
- Health check endpoint works
- No uncaught exceptions

**Fix:**
```bash
# Check Render logs
# Look for startup errors
# Verify all env vars are set
```

### Issue: CORS Errors
**Check:**
- `ALLOWED_ORIGINS` includes your frontend URL
- Frontend is using correct API URL

**Fix:**
```
ALLOWED_ORIGINS=https://hashnhedge.com,https://yourdomain.com
```

## 🔐 Security Notes

### Environment Variables
- ✅ Auto-generate `ADMIN_API_KEY`, `SESSION_SECRET`, `JWT_SECRET`
- ✅ Never commit `.env` file
- ✅ Use Render's environment variable encryption

### Database
- ✅ Use internal database URL when possible
- ✅ Enable SSL (automatic with Render PostgreSQL)
- ✅ Regular backups (Render provides automatic backups)

### CORS
- ✅ Whitelist only your domains
- ✅ Don't use `*` in production
