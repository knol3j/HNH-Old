# HashNHedge Deployment Guide

## Overview

This guide covers deploying HashNHedge to Render and Railway platforms. The project consists of three main services:

1. **Main API Server** - Unified backend API (api/server-unified.js)
2. **Hybrid Pool Server** - AI/Mining pool (hybrid-pool/index.js)
3. **Mobile Proof Pool** - PhoneProof mining (mobile-proof-pool/src/mobile-pool-server.js)

## 🚀 Render Deployment

### Configuration File
All services are configured in `render.yaml` at the project root.

### Services

#### 1. Main API Server (`hashnhedge-api`)
```yaml
buildCommand: npm run deploy:build
startCommand: npm run start:unified
healthCheckPath: /api/health
```

**Required Environment Variables:**
- `DATABASE_URL` - Neon/Postgres connection string (with connection pooling)
- `DATABASE_URL_UNPOOLED` - Direct Postgres connection
- `JWT_SECRET` - Min 32 characters (auto-generated by Render)
- `SESSION_SECRET` - Min 32 characters (auto-generated)
- `ADMIN_API_KEY` - Admin authentication key (auto-generated)
- `OFFICIAL_WALLET_ADDRESS` - Solana wallet for pool operations
- `ALLOWED_ORIGINS` - CORS origins (comma-separated)

**Optional Environment Variables:**
- `NEON_API_ENDPOINT` - Neon database API endpoint
- `NEXT_PUBLIC_STACK_PROJECT_ID` - Stack Auth project ID
- `NEXT_PUBLIC_STACK_PUBLISHABLE_CLIENT_KEY` - Stack Auth public key
- `STACK_SECRET_SERVER_KEY` - Stack Auth server key

#### 2. Hybrid Pool Server (`hashnhedge-pool`)
```yaml
rootDir: hybrid-pool
buildCommand: npm install --production=false
startCommand: node index.js
healthCheckPath: /health
```

**Required Environment Variables:**
- `DATABASE_URL` - PostgreSQL connection string
- `ADMIN_API_KEY` - Admin authentication key
- `POOL_FEE_AI` - AI job fee percentage (default: 0.30)
- `POOL_FEE_MINING` - Mining fee percentage (default: 0.03)
- `MIN_PAYOUT` - Minimum payout threshold (default: 0.01)

**Optional Environment Variables:**
- `AWS_ACCESS_KEY_ID` - For S3 backups
- `AWS_SECRET_ACCESS_KEY` - For S3 backups
- `AWS_REGION` - AWS region (default: us-east-1)
- `S3_BUCKET_NAME` - S3 bucket for backups
- `SENDGRID_API_KEY` - Email notifications
- `SENDGRID_FROM_EMAIL` - Sender email address

#### 3. Mobile Proof Pool (`hashnhedge-mobile-pool`)
```yaml
rootDir: mobile-proof-pool
buildCommand: npm install --production=false
startCommand: node src/mobile-pool-server.js
healthCheckPath: /api/stats
```

**Required Environment Variables:**
- `DATABASE_URL` - PostgreSQL connection string
- `POOL_ADDRESS` - Cryptocurrency pool address
- `POOL_FEE` - Pool fee percentage (default: 2)
- `MIN_PAYOUT` - Minimum payout threshold

## 🚂 Railway Deployment

### Configuration Files
- **Main Project:** `railway.toml` (root directory)
- **Hybrid Pool:** `hybrid-pool/railway.toml`

### Main API Deployment

#### Configuration (`railway.toml`)
```toml
[build]
builder = "NIXPACKS"
buildCommand = "npm install && npm run prisma:generate"

[deploy]
startCommand = "npm run start:unified"
healthcheckPath = "/api/health"
```

#### Deployment Steps

1. **Install Railway CLI**
   ```bash
   npm install -g @railway/cli
   railway login
   ```

2. **Initialize Project**
   ```bash
   railway init
   railway link
   ```

3. **Set Environment Variables**
   ```bash
   # Required variables
   railway variables set DATABASE_URL="postgresql://..."
   railway variables set JWT_SECRET="your-secret-here"
   railway variables set SESSION_SECRET="your-secret-here"
   railway variables set ADMIN_API_KEY="your-api-key"
   railway variables set NODE_ENV="production"

   # Optional variables
   railway variables set ALLOWED_ORIGINS="https://hashnhedge.com"
   railway variables set OFFICIAL_WALLET_ADDRESS="your-wallet"
   ```

4. **Deploy**
   ```bash
   railway up
   ```

5. **Check Deployment**
   ```bash
   railway logs
   railway open
   ```

### Hybrid Pool Deployment

#### Configuration (`hybrid-pool/railway.toml`)
```toml
[build]
builder = "NIXPACKS"
buildCommand = "npm install --production=false"

[deploy]
startCommand = "node index.js"
healthcheckPath = "/health"
```

#### Deployment Steps

1. **Create New Service**
   ```bash
   cd hybrid-pool
   railway init
   ```

2. **Set Environment Variables**
   ```bash
   railway variables set NODE_ENV="production"
   railway variables set PORT="10000"
   railway variables set STRATUM_PORT="3333"
   railway variables set STRATUM_HOST="0.0.0.0"
   railway variables set DATABASE_URL="postgresql://..."
   railway variables set ADMIN_API_KEY="your-api-key"
   ```

3. **Deploy**
   ```bash
   railway up
   ```

## 🔍 Troubleshooting

### Common Issues

#### 1. Prisma Client Not Generated
```bash
# Error: Cannot find module '@prisma/client'
# Fix: Run prisma:generate in build command
npm run prisma:generate
```

#### 2. Database Connection Failed
```bash
# Check DATABASE_URL format
# Should be: postgresql://user:pass@host:port/dbname

# For Neon (Render):
DATABASE_URL="postgresql://user:pass@host/dbname?sslmode=require"

# For Railway:
DATABASE_URL=${{ DATABASE_URL }}  # Uses Railway's internal variable
```

#### 3. Health Check Failing
```bash
# Verify endpoints are accessible
curl https://your-app.onrender.com/api/health
curl https://your-app.railway.app/api/health

# Check logs
render logs --service hashnhedge-api
railway logs
```

#### 4. Build Timeout
```bash
# Solution: Use --production=false to include all dependencies
buildCommand: npm install --production=false
```

#### 5. Port Binding Issues
```bash
# Render: Always uses PORT environment variable
# Railway: Uses PORT or defaults to 10000

# In your app, use:
const PORT = process.env.PORT || 10000;
```
