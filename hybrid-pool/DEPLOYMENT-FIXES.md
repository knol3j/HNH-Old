# 🔧 Hybrid Pool Deployment Fixes

## Issue Summary

The hybrid-pool deployment was failing due to ADMIN_API_KEY validation errors. The application required a minimum 32-character API key for security, but default configurations were using shorter values like "change-me".

## Fixes Applied

### 1. **admin-api.js** - Relaxed Development Mode Validation

**File**: `hybrid-pool/admin-api.js:15-43`

**Changes**:
- Modified API key validation to distinguish between production and development modes
- In **production**: Enforces strict 32-character minimum requirement
- In **development**: Shows warnings but allows shorter keys for testing
- Prevents insecure default values only in production

**Code Changes**:
```javascript
// Before: Always threw error for keys < 32 characters
if (apiKey.length < 32) {
    throw new Error('ADMIN_API_KEY must be at least 32 characters');
}

// After: Environment-aware validation
const isProduction = process.env.NODE_ENV === 'production';

if (isProduction && apiKey.length < 32) {
    throw new Error('ADMIN_API_KEY must be at least 32 characters in production');
} else if (!isProduction && apiKey.length < 32) {
    console.warn('⚠️  WARNING: ADMIN_API_KEY is shorter than recommended');
}
```

### 2. **render.yaml** - Added PORT and Health Check

**File**: `render.yaml:48-88`

**Changes**:
- Added `PORT: 10000` environment variable (Render's default HTTP port)
- Added `healthCheckPath: /health` for Render's health monitoring
- Updated service description to "Hybrid Pool Server (AI/Mining)"
- Confirmed `ADMIN_API_KEY: generateValue: true` is set

### 3. **.env.example** - Updated Documentation

**File**: `hybrid-pool/.env.example:11-18`

**Changes**:
- Added clear warning about 32-character requirement for production
- Provided command to generate secure API key
- Updated default value to be 32+ characters
- Added note about development mode allowing shorter keys

### 4. **Dockerfile** - Fixed Health Check Port

**File**: `hybrid-pool/Dockerfile:23-36`

**Changes**:
- Updated health check to use `${PORT:-3334}` instead of hardcoded `3335`
- Updated EXPOSE directive to use PORT environment variable
- Added comments explaining Render's PORT=10000 configuration

## Deployment Instructions

### Render.com Deployment

1. **Environment Variables** are already configured in `render.yaml`:
   - `NODE_ENV=production`
   - `PORT=10000`
   - `ADMIN_API_KEY` (auto-generated by Render)
   - All other required variables

2. **Deploy via Render Dashboard**:
   ```bash
   # Push changes to GitHub
   git add .
   git commit -m "fix: Resolve ADMIN_API_KEY validation for hybrid-pool deployment"
   git push origin master
   ```

3. **Monitor Deployment**:
   - Render will auto-deploy when changes are pushed
   - Check logs at: https://dashboard.render.com
   - Health check endpoint: `https://hashnhedge-pool.onrender.com/health`

### Railway.app Deployment

1. **Set Environment Variables** in Railway Dashboard:
   ```bash
   NODE_ENV=production
   PORT=10000
   STRATUM_PORT=3333
   ADMIN_API_KEY=<generate-via-railway-ui>
   ```

2. **Generate ADMIN_API_KEY**:
   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

3. **Deploy**:
   ```bash
   railway up
   ```

   Or use auto-deploy from GitHub.

### Local Development

1. **Copy .env.example**:
   ```bash
   cd hybrid-pool
   cp .env.example .env
   ```

2. **Edit .env**:
   ```bash
   NODE_ENV=development
   ADMIN_API_KEY=dev-key-123  # Minimum 10 characters for dev
   ```

3. **Start Server**:
   ```bash
   npm install
   npm start
   ```

## Security Notes

⚠️ **Production Security Requirements**:
- ADMIN_API_KEY must be exactly 32+ characters
- Do not use default values like "change-me", "admin", "password"
- Generate keys using: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`
- Never commit .env files with real API keys

✅ **Development Mode**:
- Shorter API keys are allowed (minimum 10 characters)
- Warnings are shown but app continues to run
- Use NODE_ENV=development to enable relaxed mode

## Troubleshooting

### Build Still Fails

**Symptom**: "ADMIN_API_KEY must be at least 32 characters in production"

**Solution**:
1. Verify NODE_ENV=production is set
2. Check ADMIN_API_KEY in Render/Railway dashboard
3. Regenerate API key with: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`

### Health Check Fails

**Symptom**: Render shows service as unhealthy

**Solution**:
1. Verify PORT=10000 is set in environment
2. Check logs for startup errors
3. Test health endpoint manually: `curl https://hashnhedge-pool.onrender.com/health`

### Stratum Port Not Accessible

**Symptom**: Miners can't connect to port 3333

**Solution**:
1. Render's free tier only supports HTTP traffic on the PORT variable
2. Stratum requires TCP which needs a paid plan
3. For Stratum mining, deploy to Railway which supports TCP ports
