# 🚀 Deploying HashNHedge Hybrid Pool to Render

## Overview

This guide walks you through deploying the HashNHedge Hybrid Pool to Render at `https://hashnhedge-pool.onrender.com`.

## Prerequisites

- GitHub repository connected to Render
- Render account (free tier works)
- Optional: PostgreSQL database, AWS S3, SendGrid accounts

## Deployment Steps

### 1. Connect Repository to Render

1. Log in to [Render Dashboard](https://dashboard.render.com)
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository: `knol3j/HNH`
4. Select the repository

### 2. Configure Service

Use these settings in Render:

- **Name**: `hashnhedge-pool`
- **Region**: `Oregon (US West)` or closest to your users
- **Branch**: `master`
- **Root Directory**: `hybrid-pool`
- **Environment**: `Node`
- **Build Command**: `npm install`
- **Start Command**: `npm start`
- **Plan**: `Free` (or upgrade as needed)

### 3. Environment Variables

Add these environment variables in Render dashboard:

#### Required Variables:

```bash
NODE_ENV=production
STRATUM_PORT=3333
STRATUM_HOST=0.0.0.0
ADMIN_API_KEY=<auto-generated-or-custom>
```

#### Pool Fee Configuration:

```bash
POOL_FEE_AI=0.30
POOL_FEE_MINING=0.03
MIN_PAYOUT=0.01
```

#### Optional (For Full Features):

**PostgreSQL Database:**
```bash
DATABASE_URL=<render-postgres-url>
```

**AWS S3 Backups:**
```bash
AWS_ACCESS_KEY_ID=<your-key>
AWS_SECRET_ACCESS_KEY=<your-secret>
AWS_REGION=us-east-1
S3_BUCKET_NAME=hashnhedge-pool-backups
```

**SendGrid Email:**
```bash
SENDGRID_API_KEY=<your-key>
SENDGRID_FROM_EMAIL=noreply@hashnhedge.com
```

### 4. Add PostgreSQL Database (Optional but Recommended)

1. In Render dashboard, click **"New +"** → **"PostgreSQL"**
2. Name: `hashnhedge-pool-db`
3. Plan: `Free` or `Starter`
4. Click **"Create Database"**
5. Copy the **Internal Database URL**
6. Add to your web service as `DATABASE_URL` environment variable

### 5. Deploy

1. Click **"Create Web Service"**
2. Render will automatically build and deploy
3. Wait 2-5 minutes for deployment
4. Your pool will be live at: `https://hashnhedge-pool.onrender.com`

## Accessing the Pool

### Admin API Endpoints

Base URL: `https://hashnhedge-pool.onrender.com`

#### Public Endpoints (No Auth):
- `GET /health` - Health check
- `GET /metrics` - Prometheus metrics

#### Protected Endpoints (Require X-API-Key header):
- `GET /stats` - Pool statistics
- `GET /overview` - Pool overview
- `GET /workers` - List all workers
- `GET /workers/:id` - Worker details
- `GET /jobs` - Job queue status
- `GET /payments` - Payment history
- `GET /revenue` - Revenue analytics

**Example:**
```bash
curl -H "X-API-Key: your-admin-api-key" \
  https://hashnhedge-pool.onrender.com/stats
```

### Stratum Mining Connection

**Note:** Render's free tier doesn't support TCP/Stratum connections directly. For Stratum mining:

**Option 1: Use Admin API only**
- Deploy to Render for monitoring/admin dashboard
- Run Stratum server locally or on dedicated server

**Option 2: Upgrade Render Plan**
- Upgrade to paid plan with custom ports
- Configure Stratum port (3333) in Render settings

**Option 3: Use WebSocket Proxy**
- Add WebSocket-to-Stratum bridge
- Miners connect via WebSocket

## Health Checks

Render will automatically check `/health` endpoint. The pool is healthy if:
- HTTP 200 response
- JSON contains `{ "status": "healthy" }`

## Logs and Monitoring

View logs in Render dashboard:
1. Go to your service
2. Click **"Logs"** tab
3. See real-time logs from the pool

Look for:
- ✅ `Hybrid pool ready on 0.0.0.0:3334`
- 📊 Worker connections
- 💰 Revenue events
- ⚠️ Errors or warnings

## Troubleshooting

### Build Fails

**Error:** `npm install` fails
- Check `package.json` in `hybrid-pool/` directory
- Ensure Node version matches (18+)
- Check build logs for specific error

### Deploy Fails

**Error:** Port binding issue
- Ensure `PORT` env var is used (not hardcoded)
- Check `index.js` uses `process.env.PORT`

**Error:** Database connection fails
- Verify `DATABASE_URL` is set
- Check PostgreSQL database is running
- Test connection from logs

### Health Check Fails

**Error:** `/health` endpoint not responding
- Check if Admin API is starting
- Verify port configuration
- Check logs for startup errors

### No Stratum Connections

**Expected on Free Tier:** Render free tier doesn't support custom TCP ports
- Use for Admin API only
- Or upgrade to paid plan

## Scaling

### Free Tier Limits:
- 512 MB RAM
- Shared CPU
- Sleeps after 15 minutes of inactivity
- No custom TCP ports

### Upgrade for:
- **Starter ($7/month)**: 512 MB RAM, no sleep
- **Standard ($25/month)**: 2 GB RAM, dedicated resources
- **Pro ($85/month)**: 4 GB RAM, custom ports, priority support

## Manual Deployment

Force a new deployment:
1. Go to Render dashboard
2. Click your service
3. Click **"Manual Deploy"** → **"Deploy latest commit"**

Or push to GitHub (auto-deploy enabled):
```bash
git add .
git commit -m "Update hybrid pool"
git push origin master
```
